## 1. Smaller the step, better the result.
_Vibe_ coding has the _vibe_ of not knowing details. It works for one-off task, like the ones showcased on all the coding agent's marketing sites, but if you need to maintain the project, you absolutely need to know how everything works. 
Instead of prompting a long list of requirements, then send coding agent off to do whatever it wants, we need to feed it granular specifications, the smaller the step you (as you the developer and coding agent together) take on building, the better you, the developer, will be in long run.

## 2. No building without planning.
Always make the coding aganed to plan first, list all the things it'll do, you have to review first, ask for changes if it doesn't suit your need. Only make coding agent to code after you approve it's planning.

### Maybe a third, prepare for a huge consumption of tokens
All AI agents consumes lots of tokens, coding agents are one the extremists. Go with coding plan if your supplier has one. If you're using normal API for coding agent, I hope your credit card is ready.
